# `wiki-sync` — Failure Modes (Detailed Catalog)

> Supplementary material for `wiki-sync`. Not loaded at LLM runtime.

---

## Workspace state failures

### Target repo not initialized (no `CLAUDE.md` or `research-wiki.config.yaml`)

- **Action:** abort with a clear message. Suggest `wiki-init`.

### `wiki/` does not exist yet

- **Action:** generate `index/` only; skip the stale-link phase entirely. Not an error — early-lifecycle state.

### Working tree dirty during scan

- **Action:** proceed; prefix the snapshot's `## Repository` section with `⚠ Working tree dirty` listing git HEAD and uncommitted file counts. Do not refuse to write — dirty-state snapshots are still valid history, just labeled.

---

## Scanner failures

### No scanner available for a language

- **Action:** warn in the summary, skip those files, continue. Example: `- Skipped: src/experiments/*.jl (no scanner for Julia; see config wiki_sync_ignore to suppress)`.
- **Recovery:** add a per-language scanner adapter (out-of-tree); or add the path to `wiki_sync_ignore:`.

### Scanner throws a parse error on a specific file

- **Action:** log the file and error line in `## Scan errors` section. Include whatever the scanner could parse. Do not skip the whole file silently — partial parse is useful for downstream wiki-log auto-link.

### Scanner times out on a single file (e.g., huge auto-generated file)

- **Action:** skip just that file; log to `## Scan errors`. Other files scan normally.
- **Recovery suggestion:** add the file to `wiki_sync_ignore:` if auto-generated; or configure the appropriate scanner.

### `--scan-body` body link rot tokenizer fails on a specific page

- **Action:** record in `## Scan errors`. Skip the page from body link rot pass. Other passes (frontmatter stale check, reverse_refs build) still run on this page.

---

## Config failures

### `research-wiki.config.yaml` missing

- **Action:** fall back to built-in defaults. Warn: "research-wiki.config.yaml not found at target-repo root; using defaults. Run `wiki-init` to scaffold one." Do not abort.

### Unknown `sync.*` keys

- **Action:** ignore, warn on stderr.

### `--scan-body` enabled but `index/signatures.json` is empty

- **Action:** cannot resolve mentions; warn and skip body link rot pass; continue with the rest of sync.

### `--scan-body` token regex matches an ordinary word

E.g., "Forward" used as English noun, not a class name.

- **Action:** record as a body mention candidate. The implicit `[unverified]` semantic carries through to `wiki-fix-stale`, where the researcher can mark the false positive and clear it without a body edit.

---

## Cross-skill state failures

### `deep/last-scan.yaml` exists but was generated by a different tool version

- **Action:** proceed. Note in the snapshot's `## Repository` section that the "since last sync" diff may be conservative.

### `index/snapshots/` directory missing

- **Action:** create. Not an error.

### Existing wiki page has malformed frontmatter

- **Action:** skip just that page from the stale-link pass; record on stderr. Do not touch the malformed page.

---

## Output failures

### Cannot write to `index/`

- **Action:** abort. Report OS-level error. Previous `index/` state preserved.

### Cannot write `index/snapshots/sync_*.md`

- **Action:** abort. Warn the researcher that the immutable history is broken — the run did not complete. Recovery: investigate write permissions.

### Cannot append to `wiki/questions.md`

- **Action:** report failure. Stale-link flags in frontmatter are still written (they are independent edits per file). The researcher can manually copy the to-be-appended notes from stderr.

---

## Dirty-state edge cases

### Working tree has unstaged file deletions that wiki-sync would observe as "removed symbols"

- **Action:** treat as removed for the snapshot. The dirty-state warning makes clear the diff is against the working tree. Researcher who reverts the deletion will see the symbol re-appear on the next sync.

### Symbolic links in code paths

- **Action:** follow once; do not loop. Mark traversal-cycle paths in `## Scan errors`.

---

## Nag interactions

### `--no-nag` set but no stale flags exist

- **Action:** no-op. The nag would not fire anyway.

### Nag age computed against `stale_detected:` field that is malformed or in the future

- **Action:** skip that ref from age computation; log to `## Scan errors`. The frontmatter integrity check is `wiki-lint`'s responsibility, not wiki-sync's.
